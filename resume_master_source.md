# 简历母版模板（内部信息源）

> 用途：作为后续生成不同岗位定制简历子版的统一信息源。
>
> 使用原则：只保留真实、可核验、可裁剪的信息；对子版按岗位目标进行取舍和重排，不直接照搬。

## 一、重要使用规则

- **真实性优先**：所有字段都必须替换成候选人的真实信息，禁止保留未核验的示例内容。
- **敏感信息内外有别**：身份证号、详细住址、证件照片原图、内部评价等敏感字段仅保留在内部母版，默认不出现在外发 PDF 中。
- **排名与荣誉要可证明**：成绩排名、奖项层级、竞赛名次、语言成绩、论文分区等表述，只有在可核验时才可对外使用。
- **论文状态需复核**：所有 `under review`、`revised`、`accepted`、`published` 等状态在外发前都要再次确认。
- **量化结果优先**：经历描述优先写动作、结果、指标和影响，而不是泛泛描述“参与”“协助”。

## 十二、子版生成建议

### 1）技术研发 / 仿真工程版
- 保留：教育背景、强相关项目、核心技能、量化成果、代表论文或技术报告。
- 弱化：与目标岗位弱相关的学生工作、泛化荣誉、无量化结论的经历。
- 默认不写：敏感身份信息与无法证明的排名。

### 2）AI+Science / 智能工具版
- 保留：编程能力、科研问题抽象能力、自动化工具、Agent / LLM 相关实践、跨学科项目。
- 强调：把真实业务或科研需求转化为工作流、系统或工具的能力。
- 可选加入：论文、开源项目、平台化经验。

### 3）算法 / 平台研发版
- 保留：算法、数据、工程化、系统实现、性能优化、生产落地相关经历。
- 强调：问题拆解、指标提升、稳定性和协作效率。
- 弱化：过于学术但与岗位不直接相关的背景信息。

### 4）学术申请 / 博后版
- 保留：研究方向、论文、会议、项目负责人经历、实验 / 数值 / 理论方法。
- 强调：研究问题、方法贡献、学术产出与独立推进能力。
- 弱化：与申请方向无关的产品化或求职导向项目。
