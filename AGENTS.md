# AGENTS.md

## 个人默认约束
- 优先做小改动，避免不必要的重构。
- 如需新增依赖，必须先征求用户确认。
- 完成任务前，运行与改动相关的验证或编译步骤。
- 未经明确要求，不改写架构类说明文档。

## 交互要求
- 当任务需要用户补充目标岗位、投递偏好、事实信息或其他关键上下文时，优先通过提问向用户确认，再继续执行。

## 临时文件规则
- 在完成任务前，以及最终回复前，自动删除本次创建或在已触达目录中发现的临时文件。
- 临时文件包括：LaTeX 编译中间产物（`*.aux`、`*.log`、`*.out`、`*.fls`、`*.fdb_latexmk`、`*.xdv`、`*.synctex*`、`missfont.log`）、PDF 审查预览目录或产物（如 `preview`、`preview_v2`、`*-preview-*`、`*_preview-*`、`*preview*.png`、`*preview*.pdf`）、OCR 暂存目录（如 `.ocr-tmp`）以及 `.DS_Store`。
- 默认保留：`.tex`、用户接受的最终 PDF、项目根目录备份 PDF、照片、信息母版、证明材料和其他源文件。
- 如果为了排障需要暂时保留日志、预览图或其他临时诊断产物，先向用户确认。

## Skill 路由
- 如果任务是生成、更新、编译、审查或排版岗位定制的 LaTeX 简历子版，优先使用项目内 skill `make-resume-variant`：
  `skills/make-resume-variant/SKILL.md`
- 上述 skill 负责简历子版的素材来源、内容筛选、子版目录、编译、PDF 审查、一页版面控制与例外多页模式。
- 共享仓库中的默认命名约定：信息母版文件为 `resume_master_source.md`，默认证件照文件统一使用 `profile_photo.png`。

## 直接处理的任务
- 如果任务是维护母版信息源本身，例如更新默认信息母版文件 `resume_master_source.md`、整理证明材料、补充课程/成绩、修正基础事实信息，可直接在项目中处理，不必重复走简历子版 skill 的完整流程。
